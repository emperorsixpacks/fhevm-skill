# ERC-7984 Confidential Token Standard Reference

> Load when building confidential tokens (encrypted balances, private transfers).

---

## What is ERC-7984?

ERC-7984 is a **new token standard** for confidential tokens. It is NOT
a modified ERC-20 — it's a distinct standard designed for on-chain privacy.

| Aspect              | ERC-20                        | ERC-7984                              |
|---------------------|-------------------------------|---------------------------------------|
| Balance type        | `uint256` (public)            | `euint64` (encrypted)                 |
| Transfer amount     | `uint256` (public)            | `euint64` (encrypted)                 |
| Total supply        | Public                        | Public (by convention)                |
| `balanceOf`         | Returns plaintext             | Returns encrypted handle              |
| `transfer`          | Takes plaintext amount        | Takes encrypted amount + proof        |
| Failure behavior    | Reverts on insufficient funds | Silent (transfers 0 if insufficient)  |

## OpenZeppelin implementation

The official implementation is in `@openzeppelin/confidential-contracts`:

```bash
npm install @openzeppelin/confidential-contracts
```

### Basic ERC-7984 token
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.27;

import {Ownable2Step, Ownable} from "@openzeppelin/contracts/access/Ownable2Step.sol";
import {FHE, externalEuint64, euint64} from "@fhevm/solidity/lib/FHE.sol";
import {ZamaEthereumConfig} from "@fhevm/solidity/config/ZamaConfig.sol";
import {ERC7984} from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";

contract MyConfidentialToken is ZamaEthereumConfig, ERC7984, Ownable2Step {

    constructor(
        address owner,
        uint64 initialSupply,
        string memory name_,
        string memory symbol_,
        string memory contractURI_
    ) ERC7984(name_, symbol_, contractURI_) Ownable(owner) {
        _mint(owner, FHE.asEuint64(initialSupply));
    }

    /// @notice Mint with plaintext amount (only owner)
    function mint(address to, uint64 amount) external onlyOwner {
        _mint(to, FHE.asEuint64(amount));
    }

    /// @notice Mint with encrypted amount (only owner)
    function confidentialMint(
        address to,
        externalEuint64 encryptedAmount,
        bytes calldata inputProof
    ) external onlyOwner returns (euint64) {
        return _mint(to, FHE.fromExternal(encryptedAmount, inputProof));
    }

    /// @notice Burn with plaintext amount (only owner)
    function burn(address from, uint64 amount) external onlyOwner {
        _burn(from, FHE.asEuint64(amount));
    }

    /// @notice Override to grant owner access to total supply
    function _update(
        address from,
        address to,
        euint64 amount
    ) internal virtual override returns (euint64 transferred) {
        transferred = super._update(from, to, amount);
        FHE.allow(confidentialTotalSupply(), owner());
    }
}
```

## Key functions in ERC-7984

### Reading balances
```solidity
// Returns encrypted handle (bytes32), NOT the actual balance
function confidentialBalanceOf(address account) external view returns (euint64);

// Returns encrypted total supply handle
function confidentialTotalSupply() external view returns (euint64);
```

### Transfers
```solidity
// Transfer with encrypted amount
function confidentialTransfer(
    address to,
    externalEuint64 encryptedAmount,
    bytes calldata inputProof
) external returns (bool);

// Transfer using already-encrypted handle
function confidentialTransfer(address to, euint64 amount) external returns (bool);

// Approve + transferFrom pattern also supported
function confidentialApprove(address spender, externalEuint64 amount, bytes calldata proof) external;
function confidentialTransferFrom(address from, address to, euint64 amount) external;
```

## Silent failure pattern

ERC-7984 transfers do NOT revert on insufficient balance.
Instead, they transfer 0:

```solidity
// Inside the transfer implementation
ebool canTransfer = FHE.ge(senderBalance, amount);
euint64 transferAmount = FHE.select(canTransfer, amount, FHE.asEuint64(0));
// Proceeds with transferAmount (which is 0 if insufficient)
```

**Why**: A revert would leak information (observer knows the balance is less
than the amount). Silent failure preserves confidentiality.

## Wrap / Unwrap (ERC-20 ↔ ERC-7984)

If you need interop with regular ERC-20 tokens:

```solidity
import {ERC7984Wrapper} from "@openzeppelin/confidential-contracts/token/ERC7984/extensions/ERC7984Wrapper.sol";

contract WrappedToken is ZamaEthereumConfig, ERC7984Wrapper, Ownable {
    constructor(
        IERC20 underlying,
        string memory name,
        string memory symbol,
        string memory uri
    ) ERC7984(name, symbol, uri) ERC7984Wrapper(underlying) Ownable(msg.sender) {}
}
```

Users can:
1. `wrap(amount)` — deposit ERC-20, receive ERC-7984 (encrypted balance)
2. `unwrap(amount)` — burn ERC-7984, receive ERC-20 (plaintext balance)

## Common mistakes

### WRONG: Treating ERC-7984 like ERC-20
```solidity
// WRONG — balanceOf doesn't exist the same way
uint256 balance = token.balanceOf(user);  // Returns encrypted handle, not uint256
```

### WRONG: Expecting revert on insufficient balance
```solidity
// WRONG — this won't revert
vm.expectRevert();
token.confidentialTransfer(to, tooMuchAmount);  // Silently transfers 0
```

### WRONG: Forgetting ACL in _update override
```solidity
// Missing FHE.allow in _update means nobody can decrypt their new balance
function _update(address from, address to, euint64 amount)
    internal override returns (euint64) {
    return super._update(from, to, amount);
    // Forgot: FHE.allow(confidentialTotalSupply(), owner());
}
```
